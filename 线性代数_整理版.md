# 线性代数笔记 - 几何直觉版

> **核心理念**: 线性代数的本质是几何理解,把抽象公式解读成几何概念
矩阵本质上是对n维空间中线性变换的描述，代表一套完整的空间变形规则。为了量化两个向量之间的空间位置关系，我们定义了向量点乘这一运算，它在几何上表示向量A在向量B方向上的投影强度，而在代数形式上，这个投影过程恰好可以看作一个1×n的行矩阵作为线性泛函作用于一个n维列向量，实现从高维空间到一维标量的线性变换。更深入地看，这个行向量实际上属于对偶空间，它本身就是一个"测量装置"，专门用来提取另一个向量在特定方向上的分量。因此，点乘运算将几何上的投影关系与代数上的矩阵运算完美统一起来：同一个数值既是投影的有向长度，也是坐标分量的加权求和，这种统一性不仅让我们能够用矩阵运算精确计算向量间的空间关系，更揭示了线性代数的核心思想——几何直观与代数计算是同一事物的两个侧面，而矩阵正是连接这两个世界的桥梁，它既能清晰描述空间变换的几何过程，又能提供可计算的代数形式，使得抽象的空间关系得以量化和操作
向量 ↔ 矩阵 ↔ 函数 ↔ 线性变换 ↔ 投影 都是有关联的
---

## 📐 第一部分：向量与空间基础

### 1. 二维空间的本质

想象一个无限延伸的 1×1 表格,这就是二维平面的基础。

**单位向量**:
- x轴方向: `[1, 0]` 记作 **i^**
- y轴方向: `[0, 1]` 记作 **j^**

通过单位向量乘以不同的系数,可以得到平面内的所有向量(张成空间)。

**空间变换示例**:
- **旋转90°**: 单位向量变为 `[0, 1]` 和 `[-1, 0]`
- **剪切变换**: 单位向量变为 `[1, 0]` 和 `[1, 1]`

### 2. 向量的模长(长度)

**勾股定理**: `c² = a² + b²`

**向量A的模长**: `|A| = √(x² + y²)`

这就是向量的长度,也叫"取模"。

## 🎯 第二部分：向量的点乘(Dot Product)

### 核心公式

```
向量A · 向量B = x₁x₂ + y₁y₂ = |A| × |B| × cos(θ)
```

### 几何意义:求投影

点乘计算的是**向量B在向量A上的投影长度**。

```
          B 向量
         /
        /
       /  θ 夹角
      /------
     /
    A 向量 →
-----------------------------------------
          B
         /|
        / |
       /  |  垂直线
      /   |
     /----●-------------------> A
        投影长度
```

**理解要点**:
- `cos(θ)` = 投影比例
- `|B| × cos(θ)` = B在A上的影子长度  
- `点积` = A的长度 × 影子长度

---

### 现实意义:协作程度

**点积真正求的是**: 两个向量"同向程度"的量化值

#### 生活例子

**例子1: 一起推车**
- 你往前推 → 向量A
- 我也往前推 → 向量B
- 方向完全一样 → **dot很大(正值)** → 车飞快走

**例子2: 你推我侧推**
- 方向垂直 → **dot = 0** → 我完全没帮上你,也没捣乱

**例子3: 你推我拽**
- 方向相反 → **dot负数** → 我在拖你后腿

---

### 公式拆解

```
a · b = |a| × |b| × cos(θ)
```

- `cos(θ)` 决定方向像不像:
  - `1` = 完全一样
  - `0` = 完全没关系
  - `-1` = 完全相反
- `|a|`、`|b|` 是两个向量的力度/强度

**所以点积 = 方向对齐程度 × 各自强度 = 总"协作效果"**

---

### 核心总结(记住这句)

> **点积 = 一个向量在另一个向量上的"有效贡献值"**
> - 同向 → 贡献大(正值)
> - 垂直 → 没贡献(0)
> - 反向 → 负贡献(拖后腿)

---

### 游戏开发中的应用

所有"方向相关"逻辑都靠点积:

1. **光照**: 法线和灯光方向齐不齐 → 决定亮度
2. **视角**: 敌人在不在你背后 → dot为负
3. **移动**: 角色是否顺着斜坡走
4. **爆炸**: 谁在冲击波正面,谁在侧面
5. **AI**: 怪物看没看到你

**点积不求长度、不求距离、不求位置,只求: 两个方向搭不搭、齐不齐、顺不顺。**

## 📏 第三部分：向量归一化

### 什么是归一化?

**归一化 = 把向量长度强行缩成1,方向保持不变**

### 公式

对于向量A(x, y):

```
归一化向量A = (x/√(x²+y²), y/√(x²+y²))
```

**示例**: 向量A(3, 4)
- 模长 = √(3² + 4²) = 5
- 归一化后 = (3/5, 4/5) = (0.6, 0.8)

![向量归一化](Image\归一化.png)

---

### 本质理解

归一化就是**按统一比例尺缩放**,只保留方向信息。

**重要**: 归一化后的向量可以用来表示纯方向。

---

### UE开发中的应用

1. **判断敌人是否在视野内**
2. **光照计算**:
   - 漫反射
   - 菲涅尔效果
   - 边缘光计算
3. **求夹角**

## 🧮 第四部分：三角函数速查

### 基础定义

```
sin(θ) = 对边/斜边 = y坐标
       前后: 1 → 0 → -1 (抬头 → 平视 → 低头)

cos(θ) = 邻边/斜边 = x坐标  
       左右: 1 → 0 → -1 (右边 → 不转身 → 左边)

tan(θ) = 对边/邻边 = y/x
       抬头有多猛,转身有多斜
```

## 🎲 第五部分：矩阵变换(捏豆腐理论)

### 矩阵的本质

**矩阵不是乱数阵列,而是"新空间的说明书"**

矩阵的三列代表:
- **第一列**: 新空间的X轴(红轴)在旧空间里的样子
- **第二列**: 新空间的Y轴(绿轴)在旧空间里的样子  
- **第三列**: 新空间的Z轴(蓝轴)在旧空间里的样子

---

### 单位矩阵(Standard Basis)

**定义**: 对角线全为1的矩阵

**直觉**: "出厂设置的豆腐块" - 1×1×1的标准正方体,没有任何拉伸或扭曲

---

### 矩阵变换的几何意义

> **矩阵 = 空间搬运指令集**

矩阵通过记录每份原始能量在垂直坐标系中的"投影贡献",告诉向量:
- 在新世界里该向右挪几步
- 向前走多远
- 向上爬多高

**为什么用"往右、往前、往上"描述最详细?**  
因为这三个方向互相垂直,构成了最清晰的坐标系。

---

### 记忆钩子:"跟随空间一起走"

模型上的每个点(旧坐标)都住在豆腐块里。  
豆腐块被矩阵捏歪了,居民必须跟着搬家。

## ✖️ 第六部分：向量叉乘(Cross Product)

### 核心作用

**叉乘用来找垂直方向**

### 记忆钩子

> "两线定一面,叉乘定一针"

找到那根唯一的垂直长针,就抓住了平面的灵魂(朝向)。

### 典型应用

- 求墙面法线
- 定灯光朝向
- 构建局部坐标系
