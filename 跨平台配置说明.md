# C++ 跨平台开发配置（Windows/Mac）

## 问题原因

从Mac远程仓库拉取代码后，在Windows上编译失败：

1. **C++标准不匹配**：代码使用C++20特性（`requires`、`concept`），但tasks.json没有指定`-std=c++20`
2. **VSCode配置冲突**：c_cpp_properties.json只有Mac配置，Windows IntelliSense无法正常工作
3. **编译产物冲突**：Mac生成的`Main`和Windows的`Main.exe`可能被提交到Git

---

## 解决方案

### 1. 修复tasks.json - 添加C++20标准

在 `.vscode/tasks.json` 的 `args` 中添加：

```json
"args": [
    "-fdiagnostics-color=always",
    "-g",
    "-std=c++20",  // ← 新增这一行
    "${file}",
    "-o",
    "${fileDirname}\\${fileBasenameNoExtension}.exe"
]
```

### 2. 配置双平台IntelliSense

`.vscode/c_cpp_properties.json` 现在包含Win32和Mac两套配置：

```json
{
    "configurations": [
        {
            "name": "Win32",
            "compilerPath": "C:\\msys64\\ucrt64\\bin\\g++.exe",
            "cppStandard": "c++20",
            "intelliSenseMode": "windows-gcc-x64"
        },
        {
            "name": "Mac",
            "compilerPath": "/usr/bin/g++",
            "cppStandard": "c++20",
            "intelliSenseMode": "macos-clang-arm64"
        }
    ]
}
```

VSCode会根据当前操作系统自动选择对应配置。

### 3. 更新.gitignore - 忽略编译产物

```gitignore
# Windows编译产物
*.exe
*.obj
*.dll
*.lib
*.pdb

# Mac/Linux编译产物
*.out
Main
*.app
*.dylib
*.so
*.dSYM/

# VSCode个人设置（不要提交）
.vscode/settings.json
.vscode/launch.json

# CMake生成文件
build/
cmake-build-*/
CMakeFiles/
CMakeCache.txt
```

## 跨平台编译方法

### 方法1：直接使用g++（推荐，简单）

#### Windows:
```bash
g++ Main.cpp -o Main.exe -std=c++20
Main.exe
```

#### Mac:
```bash
g++ Main.cpp -o Main -std=c++20
./Main
```

### 方法2：使用CMake（可选，更专业）

项目已包含 `CMakeLists.txt`，支持跨平台编译。

#### Windows编译:
```bash
mkdir build
cd build
cmake .. -G "MinGW Makefiles"
cmake --build .
./Main.exe
```

#### Mac编译:
```bash
mkdir build
cd build
cmake ..
make
./Main
```

## 日常工作流程

### 在Windows上开发：
1. 拉取代码：`git pull`
2. 编译：按 `Ctrl+Shift+B`（VSCode）或手动运行 `g++ Main.cpp -o Main.exe -std=c++20`
3. 运行：`./Main.exe`
4. 提交：`git add` → `git commit` → `git push`

### 在Mac上开发：
1. 拉取代码：`git pull`
2. 编译：`g++ Main.cpp -o Main -std=c++20`
3. 运行：`./Main`
4. 提交：`git add` → `git commit` → `git push`

## 常见问题

### Q1: 为什么Mac上没有tasks.json配置？
A: 因为Mac使用的是Clang编译器，路径和参数可能不同。Mac用户需要自己配置tasks.json，或者直接在终端使用g++命令。

### Q2: 如果没有安装CMake怎么办？
A: 不影响！直接使用g++命令编译即可。CMake只是可选的构建工具。

### Q3: 为什么要用C++20？
A: 你的代码使用了C++20的新特性（`requires`、`concept`等），必须用C++20标准编译。

### Q4: 可以用C++17或C++11吗？
A: 不行，代码中的`requires`、`concept`、`std::floating_point`等是C++20独有的特性。
