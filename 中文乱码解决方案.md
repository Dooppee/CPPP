# Windows 中文乱码解决方案

## 问题原因

Windows终端默认使用GBK编码，而C++源文件通常是UTF-8编码，导致中文输出乱码。

**你的情况**：在PowerShell中运行 `./Main.exe` 出现乱码，但在Git Bash中正常。

---

## 快速解决（3种方法任选）

### 方法1：在PowerShell运行前先执行（推荐）

```powershell
chcp 65001
./Main.exe
```

### 方法2：使用Git Bash运行（最简单）

Git Bash默认UTF-8，不会乱码。在VSCode中按 `` Ctrl+` `` 打开终端，选择Git Bash。

### 方法3：永久设置PowerShell为UTF-8

1. 打开PowerShell配置文件：
```powershell
notepad $PROFILE
```

2. 如果提示文件不存在，先创建：
```powershell
New-Item -Path $PROFILE -Type File -Force
notepad $PROFILE
```

3. 添加以下内容并保存：
```powershell
chcp 65001 > $null
[Console]::OutputEncoding = [System.Text.Encoding]::UTF8
```

4. 重启PowerShell，以后就不会乱码了

---

### 方案2：在代码中设置控制台编码（最可靠）

在 `Main.cpp` 的 `main()` 函数开头添加：

```cpp
#include <iostream>
#include <windows.h>  // Windows专用

int main()
{
    // Windows下设置控制台为UTF-8
    #ifdef _WIN32
        SetConsoleOutputCP(65001);  // 输出UTF-8
        SetConsoleCP(65001);        // 输入UTF-8
    #endif
    
    // 你的代码...
    std::cout << "中文测试" << std::endl;
    
    return 0;
}
```

**优点**：
- 跨终端通用（CMD、PowerShell、Git Bash都有效）
- 不需要每次手动设置
- 只在Windows下生效，Mac不受影响

---

### 方案3：使用宽字符输出（跨平台）

```cpp
#include <iostream>
#include <locale>

int main()
{
    // 设置本地化
    std::locale::global(std::locale(""));
    std::wcout.imbue(std::locale());
    
    // 使用宽字符
    std::wcout << L"中文测试" << std::endl;
    
    return 0;
}
```

**缺点**：需要把所有字符串改成宽字符（`L"..."`）

---

### 方案4：修改VSCode终端默认编码

在 `.vscode/settings.json` 中添加：

```json
{
    "terminal.integrated.defaultProfile.windows": "Git Bash",
    "terminal.integrated.env.windows": {
        "LANG": "zh_CN.UTF-8"
    }
}
```

---

## 推荐做法

**最简单有效**：在代码中添加Windows编码设置（方案2）

```cpp
int main()
{
    #ifdef _WIN32
        SetConsoleOutputCP(65001);
        SetConsoleCP(65001);
    #endif
    
    // 你的代码...
}
```

这样：
- ✅ 一次设置，永久有效
- ✅ 跨终端通用
- ✅ 不影响Mac编译（`#ifdef _WIN32`只在Windows生效）
- ✅ 不需要记住命令

## 为什么Git Bash显示正常？

Git Bash默认使用UTF-8编码，所以你当前看到的输出是正常的。但如果在CMD或PowerShell中运行，可能会乱码。

添加代码中的编码设置可以确保在所有终端都正常显示。
